## 金额口径：入表金额取「价税合计(含税)」——最易错的一点
增值税电子发票上有两个数：**金额(不含税)** 与 **价税合计(含税，常标"(小写)¥"或"价税合计")**。
**报销入表一律取价税合计(含税)**，不要取不含税的"金额"。
- 例：某酒店发票「金额 ¥568.54、税额 ¥34.11、价税合计 ¥602.65」→ 入表 **602.65**，不是 568.54。
- 铁路电子客票：取「票价」即为应报金额(已含税)；**退票**取「退票费」(金额很小，如 ¥3)，不是票价。
- 机票电子发票：取价税合计(含票价+基建+燃油+服务费的含税合计)。
取错口径财务会按差额退回。提取时把"金额/税额/价税合计"都记进 `extracted/`，入表用价税合计。

## 订阅/会员：包年(连续包年)按月摊销
当公司按**月**报销订阅类费用(常见于 AI 工具、软件会员)时，**年付/连续包年要摊销到月**，只报当月那一份：
- **月度订阅**：当月票据金额全额入表(如 Suno $10/月、Claude Max $100/月)。
- **包年/连续包年**：月摊销额 = **年费 ÷ 12**，只报对应月份。例：即梦连续包年 ¥2598.88 → 每月 2598.88÷12 = **¥216.57**，5 月报这一份。
- **补报往月**：若某月忘了报，可在本次一起补上(各月一行，注明月份)。例：4 月那一份忘报 → 本次补 4 月 + 报 5 月，共 2×¥216.57。
- 摊销/补报都要写一条额外说明，把"年费、÷12、报哪几个月、各月金额、合计"算清楚(见 `references/额外说明写法.md`)。
- 摊销月数/起算口径(按购买日还是自然月)拿不准时，列存疑问用户，不要自定。

## 关键区分：交通费(J，差旅) vs 市内交通费(Q)
- **J 差旅交通费**：出差期间发生的「车站/机场往返 + 出差目的地的市内交通」。属于差旅费大类。
- **Q 市内交通费**：非出差状态下，日常工作在常驻城市内的打车/地铁/公交。
- 判据：看这趟消费是否处在一次**出差行程**内。拿不准时，按自主程度设置——默认列入存疑清单问用户。

## 事由(C 列)必须写够的要素 —— 来自模板表头批注，写不够财务有权退回
| 费用列 | 事由必须包含 |
|---|---|
| H 火车票/机票 | 事由 + 起止地点 |
| I 住宿费 | 出差事由(住宿城市、晚数) |
| J 交通费(差旅) | 起止地点 |
| K 出差补助 | 实际出差天数(注意：仅当往返票均个人自购时才随票申请；行政代订的由行政统一发放) |
| L 办公费 | 实际费用项目(如办公用品/资料费) |
| M 福利费 | 团建须注明团建人数 |
| N 工作餐 | 用餐时间 + 用餐人数(因统一工作安排误餐) |
| O 研发物料 | 物料名称(单价须<600元) |
| P 招待费 | 客户/合作方 + 业务活动目的 + 招待人数 |
| Q 市内交通费 | 事由 + 起止地点 |
| R 会议费/培训费 | 会议/培训名称(内部或外部) |
| S 其他 | 在事由处详细列明具体事宜 |
| G 所属项目编码 | 研发/销售项目须正确填写；无项目归属填「无」 |

回填事由时，按上表把要素补齐。要素缺失而材料里又没有的，列入存疑清单向用户索取（例如招待人数、出差天数），不要臆造。
