# 归类、命名与台账规范

## 一、三个材料桶的判据(有票 / 无票 / 辅助材料)

判一份材料进哪个桶，问一句：**它能不能作为这笔消费的「主证据」，且是否带正式票据？**

- **有票**：带正式票据，能独立作为报销主证据。包括增值税发票(纸质/电子)、财政电子票据、火车票/行程单/登机牌+航空行程单、出租车发票、定额发票等。
- **无票**：消费真实发生，但**拿不到正式发票**，只有一份能当主证据的消费记录。例如：境外消费的收据(receipt)、非发票的收据/小票、银行/信用卡账单中的该笔流水、商户开具的非税收据。
- **辅助材料**：本身**不能**独立证明消费，只能侧面佐证。例如：下单截图、订单详情截图、支付成功截图、App 行程记录截图、微信/钉钉沟通确认截图。

关键认识：**同一笔消费可能同时产生分属不同桶的材料**。例如一次打车 = 出租车发票(有票) + 滴滴支付截图(辅助材料)。这两份要按「同事件」处理(见第三节)。

> 另有两个**非证据**的特殊目录(不参与本节判桶，单独管理)：
> - **替票/**：客户(可选)提供的、其他消费的正规发票池，在金额上大致冲抵「无票」项给财务(见 SKILL.md 第 9 步)。**替票与无票非一一对应**：替票**不转入「有票」**、无票也**不改判为「有票」**、不计入台账金额，只单独附给财务。
> - **待核实/ 无关/**：见第五节。

判桶拿不准时，按当前自主程度处理：默认列入存疑清单交用户裁定，不要硬塞。

### 1.1 只有截图、没有正式发票的真实消费，进「无票」不是「辅助材料」
这是最常见的判桶错误。判据看**这份材料是不是这笔消费目前唯一/最强的凭证**：
- 一笔消费**同时有**正式发票 + 支付/下单截图 → 发票进「有票」作主证据，截图进「辅助材料」。
- 一笔消费**只有**支付/订单/行程截图，**拿不到正式发票**(如个人滴滴/出租车、星巴克小票、境外 Uber、某些 App 缴费) → 这张截图就是主证据，进「**无票**」，不要丢进「辅助材料」。丢进辅助材料会导致这笔消费在台账里没有主证据、最终漏报或被财务质疑。
- 一笔消费既无发票也无任何可信凭证 → 进存疑清单向用户索取。
口诀：**辅助材料永远是"配角"——它旁边必须站着一个主证据(有票或无票)。落了单的截图要么升格为无票主证据，要么进存疑。**

### 1.2 退款/退票截图不是消费，不单独占一笔
"退款入账""中铁网络-退款""+12.00"这类是**冲减**，不是支出。它只用于核对/抵减对应那笔消费，不在表里单独列正数行；归入相关事件的辅助材料，并在额外说明里写明冲减逻辑。

## 二、文件命名规范

格式（下划线分隔，已核实的信息）：

```
日期_费用类型_关键信息_金额
```

- **日期**：`YYYY-MM-DD`(消费发生日，非提交日)。
- **费用类型**：用模板口径，如 火车票、住宿费、市内交通、工作餐、招待费、会议费、其他 等。
- **关键信息**：让财务一眼看懂的最小信息。差旅类放起止地点(`北京-上海`)；餐饮/招待放人数或对象(`3人`/`General Robotics`)；办公/物料放品名。
- **金额**：`553.5元`；外币写原币种与换算，如 `USD12.5_折89.75元`。

示例：
- `2026-06-15_火车票_北京-上海_553.5元`
- `2026-06-16_市内交通_虹桥-张江往返_88.6元`
- `2026-06-17_工作餐_GeneralRobotics对接3人_260元`

非法字符(`/ \ : * ? " < > |` 及换行)一律替换为 `-`。

## 三、同一事件的多份材料

为每个「消费事件」分配一个事件号(如 `E03`)。

- **跨桶**(同一事件的材料分散在不同桶)：**主文件名保持一致**，便于财务串起来。
  - 有票/ `2026-06-16_市内交通_虹桥-张江往返_88.6元.pdf`
  - 辅助材料/ `2026-06-16_市内交通_虹桥-张江往返_88.6元.png`
- **同桶多份**(同一事件在同一个桶里有多份)：主名相同 + 语义后缀区分。
  - `..._88.6元_发票.pdf`、`..._88.6元_行程单.pdf`
  - 后缀建议：`_发票` `_行程单` `_登机牌` `_收据` `_支付截图` `_订单截图`，无语义时用 `_01` `_02`。

### 3.1 一笔消费只入表一行 —— 绝不把"发票 + 它的支付截图"算成两笔(头号大错)
**台账里每份材料一行(用于追溯)；但回填进模板/ledger 时，按"事件"合并，一个消费事件只占一行，金额取该事件最强证据(优先发票)。**
- 反例(必须避免)：火车票发票 ¥532 + 同一程的 12306 支付截图 ¥532 → 若两条都进 ledger 就把 532 报了两次。正确：一行 ¥532(取发票)，截图作辅助材料不进 ledger。
- 同一事件的多份材料金额可能略有出入(票面 vs 实付、含/不含优惠券)。**以正式发票金额为准**；差异写进额外说明。
- 一份订单截图可能覆盖**多笔**消费(如一次 12306 下单含两程 ¥533+¥118=¥651)。它佐证两个事件，但每个事件仍各自按发票独立入表，截图只作辅助。
- 重复截图(同一笔消费截了好几张支付页) → 同一事件号，挑一张作代表，其余 `_01/_02`，**不重复入表**。

### 3.2 改签票：以改签后的有效票为准，别把原票和改签票算两遍
铁路客票上「始发改签」标记表示这是改签**后**的有效票。
- 若材料里**同一程**既有原票又有改签后票(金额不同或注明改签) → **只计最终实际乘坐的那张**，另一张作辅助材料并在额外说明写明，避免重复计费。
- **金额以铁路电子客票票面为准**：购票截图与票面差几元(改签差价/手续费口径不同)是常事，**不必纠结小额差额，按票面入表即可**；只有差额较大时才在额外说明里说明并列存疑。
- 一段连续行程被拆成多张**不同区间**的票(如 苏州北→南京南→济南西→北京南)是**各自独立的乘车段**，应分别入表，不是重复，不要误删。区分"同一程的改签重复" vs "不同区间的多段"靠**起止站+车次+时间**判断；判不准列存疑。

### 3.3 企业已代付 / AA / 个人垫付：只报个人实际承担部分
截图出现"企业支付 X 元 / 个人支付 Y 元"(如滴滴企业版)时，**企业已付的 X 不可再报**(否则重复报销)，入表只取**个人实付 Y**。在额外说明按"拆分"模板写清 企业X + 个人Y = 总额，入表 Y。同理多人 AA 只报本人份额。

## 四、过程文件：文件↔信息映射台账

在 `过程文件/` 下生成 `物料台账.csv`(UTF-8-SIG，便于 Excel 直接打开)，每份**原始材料**一行：

| 列 | 含义 |
|---|---|
| 事件号 | 如 E01；同一消费的多份材料共用 |
| 原文件名 | 备份中的原名 |
| 备份路径 | 相对工作区路径 |
| 桶 | 有票/无票/辅助材料 |
| 费用类别 | 模板口径(对应某一金额列) |
| 模板列 | H–S 中的字母 |
| 日期 | YYYY-MM-DD |
| 起点 / 止点 | 差旅类填，否则留空 |
| 金额RMB | 入表金额 |
| 原币种金额 | 外币时填，如 USD 12.5 |
| 汇率 | 外币时填 |
| 事由 | 已补齐批注要素的完整事由 |
| 所属部门 / 项目编码 | 行级 |
| 新文件名 | 重命名后的名字 |
| 置信度 | 高/中/低 |
| 备注 | 合并、退费、存疑原因等 |

同时为每份材料写一个 JSON(`过程文件/extracted/<原名>.json`)，保存从材料里提取到的原始要素，便于追溯。

台账是后续回填(生成费用台账 ledger.json)和审阅报告的唯一数据源 —— 先把台账做对，再回填。

## 五、拿不准 / 疑似私人 / 与本次无关 —— 别硬塞进有票无票，用两个缓冲桶
报销只报**因公**且**与本次出差/业务相关**的支出。判不准的**先不要**塞进 `有票/`、`无票/`：

**先放 `待核实/`（拿不准，等用户拍板）**，常见：
- **生活缴费类**：水、电、燃气、物业、话费等——**要问用途**：若是**公司租房/办公场所**的水电(经确认因公)→ 记**其他(S)**(发票口径不一，事由写清"公司租房水电费"，待补发票)；若是个人住所/私人用途 → 不报。
- **个人订阅/会员**：视频/音乐/软件订阅(剪映、网盘等)——**问是不是生产/办公工具**：是(如视频剪辑生产工具)→ 记 其他(S)/办公费(L)并注明用途；纯私人 → 不报。
- **金额异常大又无正式发票的单据**、**疑似私人消费**(明显个人餐饮/购物/娱乐)：先核实真伪与归属。

**确认后移出 `待核实/`**：能报的移到 `有票/无票/辅助材料` 并入账；

**`无关/`（确认与本次无关 / 本期不报 / 跨下一期）**：如本次去韩国却出现的日本签证、跨期的第三国机票等。复制一份到 `无关/`，**不计入**本期台账与模板，审阅报告里列明。

处理原则：默认**不擅自计入、也不擅自删除**；在审阅报告「存疑清单」逐条列出(材料、金额、疑点、待用户确认"报 / 不报 / 无关")，用户裁定后再落桶。交付时 `待核实/` 尽量清空。

## 六、回填前的去重与勾稽自检(从台账 → ledger)
从 `物料台账.csv` 生成 `ledger.json` 前，按事件号汇总，逐项确认：
1. **去重**：每个事件号在 ledger 里只生成**一行**(金额取该事件主证据/发票)；辅助材料、退款冲减截图不单独生成行。
2. **冲减**：有退款/退票的事件，金额 = 实际净支出(原额 − 退款，或仅退票费)。
3. **拆分**：企业代付/AA 的事件，金额 = 个人实际承担部分。
4. **外币**：外币事件金额 = 换算后 RMB(汇率与算式写进额外说明)。
5. **勾稽**：Σ(ledger 金额) 应等于"各事件主证据金额之和"。把这个数记下来，回填后要与模板总额、各桶金额三方一致(见 SKILL.md 第 7 步)。
6. 任何"材料金额 ≠ 入表金额"的事件，必须有一条对应的额外说明 txt。
