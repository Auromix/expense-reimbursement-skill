# 额外说明写法(给财务看的 txt)

`额外说明/` 下放纯文本 txt，目的是让财务**不用追问就能看懂**金额是怎么来的。一种情形一个 txt，文件名与所涉事件/费用对应。语言朴素、把算式写全。

## 何时必须写额外说明
1. **合并**：多笔小额并成表里一行(最常见是同日多次打车合并)。
2. **部分退费 / 退票**：有退票/退款，入表金额≠票面金额(退票只计退票费)。
3. **改签**：原票与改签后票并存，需说明只计有效票、避免重复。
4. **外币换算**：USD / KRW 等折算成 RMB。
5. **拆分**：一张票据涉及多个费用类别、多人 AA、或企业代付+个人垫付。
6. 任何「材料金额与入表金额对不上」的情况。

## 模板一：合并打车费
```
【合并说明 — 市内交通费】
事件号：E07
合并原因：2026-06-16 当日因客户拜访多次打车，按财务口径合并为一行填报。
明细：
  1) 09:12 酒店→虹桥站   ¥36.0   滴滴发票+支付截图
  2) 13:40 张江→客户现场 ¥28.6   出租车发票
  3) 19:05 客户现场→酒店 ¥24.0   滴滴支付截图(无发票，列辅助材料)
合并金额：36.0 + 28.6 + 24.0 = ¥88.6
入表：市内交通费(Q列) ¥88.6
```

## 模板二：退票（只计退票费，不计原票价）
```
【退票说明 — 火车票】
事件号：E02
原票：北京→上海 二等座 ¥553.5
退票情况：行程取消，全额退票，扣退票手续费 ¥27.7（原票款 ¥553.5 已退回）
实际发生金额：¥27.7（仅退票手续费，本人未乘车）
入表：火车票/机票(H列) ¥27.7
说明：原票款已退回、未乘车，只有退票手续费是实际支出，故只计 ¥27.7。
　　　若另购票实际乘坐，那张实际乘坐的票按票面单独入表（改签见模板三点五）。
```
> 口径统一：**退票**只计退票费；**改签**按改签后有效票的票面、只算一次（见模板三点五），两者都不要把原票价再加进去。

## 模板三：外币换算(USD / KRW 等通用)
```
【外币换算说明 — 差旅交通费】
事件号：E20
原始消费：2026-06-12 首尔 Uber 打车 KRW 47,900(收据见 无票/)
换算汇率：0.005226 RMB/KRW(来源：用户指定 / 票面日中国银行现汇买入价，二选一注明)
换算金额：47,900 × 0.005226 = ¥250.33
入表：差旅交通费(J列) ¥250.33，币种栏仍标 RMB(已折算)
```
> 多笔同币种外币(如整趟韩国 12 单 Uber)可在同一份 txt 里列明细逐笔换算再合计，
> 并在台账各行"原币种金额/汇率"列填好。汇率未定时**先列存疑问用户**，不要自己拍一个。

## 模板三点五：改签(去重 + 差价)
```
【改签说明 — 火车票】
事件号：E04
原票：2026-05-25 苏州北→南京南 G1813 ¥103.0(注明"始发改签")
说明：本程为改签后的有效票，实际乘坐此车次；原购票记录(支付截图 ¥106)见 辅助材料/，
      二者为同一程，仅入表改签后有效票 ¥103.0，不重复计费。
入表：火车票/机票(H列) ¥103.0
```

## 模板三点六：企业代付 / 个人垫付拆分
```
【拆分说明 — 市内/差旅交通】
事件号：E12
整单：滴滴企业版 苏州公司→苏州北站，合计 ¥86.0，优惠券 −¥9.9，实付 ¥76.10
支付拆分：企业支付 ¥10.16(公司已付，不报) + 个人垫付 ¥65.94
入表：差旅交通费(J列) ¥65.94(仅本人垫付部分)
说明：企业已付部分不再报销，避免重复。原始为支付截图(无正式发票)，按"无票"主证据处理。
```

## 模板四：拆分/AA
```
【拆分说明 — 工作餐】
事件号：E05
整单金额：¥780(发票一张，见 有票/)
参与：与 General Robotics 技术对接，共 3 人
本人报销份额：¥260(整单 ÷ 3，按公司 AA 规则)
入表：工作餐(N列) ¥260
说明：其余两位参与人各自报销，避免重复。
```

写完后，在 `物料台账.csv` 对应行的「备注」里注明「见 额外说明/<txt名>」，形成闭环。
